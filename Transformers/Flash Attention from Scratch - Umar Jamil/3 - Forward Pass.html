Flash Attention Algorithm (for a single row):

  1. In a single loop (1 loop over row of SSS)
    • Compute the row-max value of SSS.
    • Update previous numerator partial sum and denominator normalization factor with new row-max
  2. Divide sum by denominator, results in a single scalar.

This algorithm can be batched:

  • We're computing multiple rows of SSS of batch size B0B0B0 at once, resulting a vector of size B0B0B0.
  • Other threads will compute other rows of SSS in parallel batches.

Why can we do this in a single loop while online softmax requires 2 loops?

The flash attention algorithm uses a reduction operation (multplies by VVV), so we can just keep adding our partial results to get to the final output.

Space complexity analysis on flash attention vs standard attention?

We know that Q,K,V∈Rn×dQ, K, V \in \mathbb{R}^{n \times d}Q,K,V∈Rn×d

S=QK⊤∈RN×NS = QK^\top \in \mathbb{R}^{N \times N} S=QK⊤∈RN×N

Standard attention: O(N2d)O(N^2d)O(N2d)

  • Each query and key is of size ddd
  • There are NNN queries and NNN keys
    • So there are N2N^2N2 attention scores
  • So the space complexity is O(N2⋅d)O(N^2 \cdot d)O(N2⋅d)

Flash attention: O(Nd)O(Nd)O(Nd)

  • Each query and key is of size ddd
  • There are NNN queries and NNN keys
    • But we compute row-wise across the attention matrix
    • So we only store 1 scalar value for each query
  • So the space complexity is O(N⋅d)O(N \cdot d)O(N⋅d)

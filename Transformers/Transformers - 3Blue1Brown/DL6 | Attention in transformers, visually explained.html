What is a query vector in attention?

  • Query vector: asks a question to the embedding vector
    • maps the embedding to a 128 dimensional space that encodes a question (with both position and token info)
    • Ex: "Are there adjectives in preceding positions?"
  • Qi=WQ⋅EiQ_i = W_{Q} \cdot E_iQi​=WQ​⋅Ei​

What is a key vector in attention?

  • Key vector: answers the question asked by the query vector
    • maps the embedding to a 128 dimensional space that encodes an answer (with both position and token info)
    • Ex: "I'm an adjective!"
  • Ki=WK⋅EiK_i = W_{K} \cdot E_iKi​=WK​⋅Ei​

How are query and key vectors used in attention?

  • For a given query vector of an embedding, which token's key vectors attend to the token's query vector?
  • The dot product of the query and key vectors is used to calculate the attention score
  • Dot product tells how how similar are the query and key vectors
    • Does the key answer the question asked by the query?
  • Both the key and query vector are in the same 128-dimensional space
  • Compute dot product of each key-query pair
    • If dot product is high, then key vector "attends to" the query vector -> large positive number
      • This implies that in 128 dim space, key and query vector point in the same direction as shown in above image

What is the attention pattern?

  • Tells us how much each token should "attend to" each other token in the context (s.t. future tokens don't attend to past tokens)
  • Let's say we have context size NNN, so we have NNN tokens, and NNN embeddings EiE_iEi​ for each token
  • For each token EiE_iEi​, we calculate a query vector QiQ_iQi​ and a key vector KiK_iKi​
  • We calculate the attention score SijS_{ij}Sij​ between each pair of embeddings EiE_iEi​ and EjE_jEj​ in the context (Cartesian product)
  • The attention score SijS_{ij}Sij​ tells us how much token EjE_jEj​ should "attend to" token EiE_iEi​
  • The attention pattern is a matrix of attention scores SijS_{ij}Sij​ for all pairs of embeddings EiE_iEi​ and EjE_jEj​ in the context

EiE_iEi​ and ViV_iVi​ are the same dims, doesn't it effectively do the same thing?

The key idea of using value vectors (ViV_iVi​) instead of directly using EiE_iEi​ in the attention mechanism lies in adding flexibility and expressiveness to the model. Let me break it down:

Why use ViV_iVi​ instead of EiE_iEi​?

  1. Learned Transformations:

    • The value vector Vi=WV⋅EiV_i = W_V \cdot E_iVi​=WV​⋅Ei​ is a learned transformation of the original embedding EiE_iEi​. The weight matrix WVW_VWV​ allows the model to decide what specific information from EiE_iEi​ should be shared with other embeddings in the context.
    • In essence, WVW_VWV​ enables the model to project EiE_iEi​ into a space that is more useful for updating the other embeddings EjE_jEj​.

  2. Separation of Concerns:

    • The embeddings EiE_iEi​ are initially transformed into queries (QjQ_jQj​) and keys (KiK_iKi​) to compute attention weights, which measure how much EiE_iEi​ contributes to updating EjE_jEj​.
    • The value vectors ViV_iVi​, on the other hand, determine what information from EiE_iEi​ is actually shared with EjE_jEj​. This separation of "importance" (attention weights) and "content" (value vectors) improves the model's ability to learn meaningful representations.

  3. Dimensional Flexibility:

    • While EiE_iEi​ and ViV_iVi​ often have the same dimensions in practice, they don’t have to. By using WVW_VWV​, the model can project the embedding into a different dimension if needed, allowing for more flexibility in the design.

  4. Better Generalization:

    • Directly using EiE_iEi​ would limit the model’s capacity to learn nuanced relationships. By introducing ViV_iVi​, the model has an additional degree of freedom to specialize the information being propagated.

Why Not Directly Use EiE_iEi​?

If you directly used EiE_iEi​ in place of ViV_iVi​:

  • EiE_iEi​ carries all the information in the original embedding, without any filtering or transformation specific to the attention computation.
  • The model might struggle to learn effectively since EiE_iEi​ includes both useful and irrelevant features for updating EjE_jEj​. By transforming EiE_iEi​ into ViV_iVi​, the model can emphasize or suppress specific parts of EiE_iEi​ based on what is most relevant.

Intuition Behind ViV_iVi​:

Think of EiE_iEi​ as raw information, and ViV_iVi​ as the "useful summary" of that information tailored to the current task. The attention weight determines how much of this summary ViV_iVi​ contributes to updating EjE_jEj​.

If we just used EiE_iEi​, we’d be treating all parts of the embedding equally, which reduces the model’s ability to focus on the most relevant features during the update. By using ViV_iVi​, we allow the model to "filter" the raw embedding into a form that's optimized for attention.

Issue with Large Variance

As ddd (the dimension of the vectors) increases, the variance of the dot product grows linearly with ddd. This large variance causes the values of the dot product to spread out significantly, leading to two issues in the softmax function:

  1. Exponential Growth: The softmax function is sensitive to large inputs, as it involves exp⁡(input)\exp(\text{input})exp(input). Large values of the dot product result in very sharp probabilities, making the attention distribution overly concentrated.

  2. Numerical Stability: The exponential function can cause numerical instability if the inputs are very large.

Summary

Incorporating the scaling factor d\sqrt{d}d​ ensures that the variance of the dot product remains constant regardless of the dimensionality ddd, preventing the attention mechanism from being biased toward extremely sharp or flat distributions.

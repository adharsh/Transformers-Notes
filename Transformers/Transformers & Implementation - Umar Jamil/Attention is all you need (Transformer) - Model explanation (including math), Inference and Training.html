How does an Recurrent Neural Network (RNN) work at a high level?

  • At each time step ttt:
    • Inputs: xtx_txt​ and the hidden state ht−1h_{t-1}ht−1​ from the previous time step
    • Outputs: yty_tyt​ and the new hidden state hth_tht​
  • If we have NNN tokens, we need to run the RNN for NNN time steps
    • NNN sequence input
    • NNN sequence output

What are the problems with RNNs?

  • Slow computation for long sequences
    • Cannot be parallelized due to sequential nature
  • Difficulty in accessing information from earlier time steps
    • Information from earlier time steps is diluted as it is passed through the chain of RNN cells
  • Vanishing or exploding gradients
    • Gradients tend to vanish or explode as they are backpropagated through time

Vanishing gradients

  • The longer the chain, the more the gradients either vanish or explode
    • If chain of gradients is ∈(−1,1)  ⟹  \in (-1, 1) \implies∈(−1,1)⟹ vanishing gradients
    • If chain of gradients is ∈(−∞,−1)∪(1,∞)  ⟹  \in (-\infty, -1) \cup (1, \infty) \implies∈(−∞,−1)∪(1,∞)⟹ exploding gradients

What is position encoding and how is it calculated?

  • Each word should carry positional information
  • Model should treat nearby words as "close" and distant words as "distant"
  • Positional encoding is calculated once and added to the embeddings
  1. Convert sentence to tokens TiT_iTi​
  2. Convert tokens to embeddings using WE⋅TiW_E \cdot T_iWE​⋅Ti​
  3. Add positional encoding to embeddings using Ei=WE⋅Ti+PiE_i = W_E \cdot T_i + P_iEi​=WE​⋅Ti​+Pi​

  • kkk: position in the sequence
    • Range: [0,N−1][0, N-1][0,N−1]
  • iii: dimension of the positional encoding
    • Range: [0,dmodel−1][0, d_{model}-1][0,dmodel​−1]
  • For each position kkk and each dimension iii of the positional encoding, we calculate a value using sine and cosine functions
  • The frequency of the sine and cosine functions increases exponentially with the dimension iii
